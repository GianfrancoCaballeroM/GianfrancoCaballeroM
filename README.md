### 🗃️ Mezzanine NoSQL — Advanced MongoDB Architecture Demo
> *A focused backend prototype demonstrating production-grade NoSQL patterns, built as the MongoDB counterpart to the main Mezzanine system.*

| | |
|---|---|
| **Context** | Bases de Datos II · Academic Project |
| **Stack** | MongoDB Atlas · Node.js/Express · React/Vite · Mongoose |

**Advanced MongoDB Patterns Implemented:**

| Pattern | Implementation |
|---|---|
| **Capped Collections** | `auditoria_logs` initialized programmatically at startup — `{ capped: true, size: 1048576, max: 1000 }` — for high-throughput circular log writes |
| **TTL Indexes** | Auto-expiry index on `fecha` field (`expires: 2592000`) — logs older than 30 days are deleted automatically by MongoDB Atlas |
| **Native DB Views** | `v_reservas_hoy` view built directly on the `reservas` collection using `$dateTrunc`, `$dateAdd`, and `$$NOW` — date logic runs at DB level, not in Node.js |
| **Aggregation Pipelines** | `$match → $group → $sort` pipeline on `Reserva` for real-time hourly customer density analytics |
| **Compound & Ranked Indexes** | `{ fecha_reserva: 1, estado: 1 }` on reservations; descending `total_reservas_historicas: -1` for VIP client reports; unique index on `email` |
| **Mongoose Triggers** | `post('save')` atomically increments `$inc` on the referenced client; `post('findOneAndUpdate')` auto-generates audit log entries on state changes |
| **Embedded Documents** | `NotaStaffSchema` nested inside `clientes` as subdocument array — avoids relational JOINs entirely |
